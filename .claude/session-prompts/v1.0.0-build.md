# v1.0.0 Build Session — Foundation & Core Skills

## Context
Salesforce Workspace is a multi-project workspace for Salesforce consulting. Read
`CLAUDE.md` for full project context. This is the first release in the Consultant
Acceleration milestone — it builds the workflow engine foundation and all core standalone
skills that playbooks will later orchestrate.

No prior releases exist. This is greenfield.

## Release Setup
```
git checkout -b release/v1.0.0 main
```

## Milestone
`v1.0.0` — 10 issues, 6 parallel lanes (A-F). Each lane creates files in completely
different directories. Zero file overlap between lanes.

**IMPORTANT**: CLAUDE.md updates are handled by the final step AFTER all agents complete.
No agent should modify CLAUDE.md during their work.

### Agent C: Discovery & Analysis Skills
**Scope:** `.claude/skills/sf-discovery/`, `.claude/skills/sf-gap-analysis/`

**Issues (in order):**
1. **#3** — /sf-discovery — guided requirements gathering skill
   - Create `.claude/skills/sf-discovery/SKILL.md` — Skill that runs structured client intake interview via AskUserQuestion, produces requirements with acceptance criteria, outputs to file and Obsidian. Reference `.claude/consulting-assets/discovery-questions.md` for question templates (created by Agent B — if not yet available, use inline questions and note the reference for later integration).

   - Output format: structured markdown with sections for business problem, stakeholders, current state, desired state, requirements, user stories, assumptions, open questions

2. **#12** — /sf-gap-analysis — current vs desired state analysis
   - Create `.claude/skills/sf-gap-analysis/SKILL.md` — Skill that cross-references org analysis (from /sf-org-analyze) against requirements, classifies each requirement as Already Met / Partially Met / Gap, classifies gaps by type (Config / Declarative / Custom Dev / Integration / Migration)
   - Output format: gap analysis matrix table with requirement, current state, gap type, effort, notes
   - Works with or without org access (manual input fallback)

**Commit pattern:**
```

### Agent D: Design Skills
**Scope:** `.claude/skills/sf-data-model/`, `.claude/skills/sf-automation-map/`

**Issues (in order):**
1. **#4** — /sf-data-model — interactive Salesforce data model designer
   - Create `.claude/skills/sf-data-model/SKILL.md` — Skill that designs Salesforce data model from entity descriptions, validates against platform limits (800 objects, 800 fields/object, 40 lookups/object, 2 master-detail/object, 5-level relationship depth), generates Mermaid ERD, outputs object/field manifest
   - Include: standard vs custom object mapping, field type selection, junction object design, record type planning, governor limit risk assessment
   - Reference: Context7 `/damecek/salesforce-documentation-context` for limit validation

2. **#5** — /sf-automation-map — process-to-automation mapping with decision guides
   - Create `.claude/skills/sf-automation-map/SKILL.md` — Skill that maps business processes to Salesforce automation tools using architect decision guides. For each process: classify trigger type, evaluate complexity, check data volume, apply decision guide framework
   - Output: process → tool → justification → risk → decision guide reference matrix
   - Reference: `salesforce-architect-decision-guides/01-Record-Triggered-Automation.md` through `08-Data-Integration.md`
   - Include: automation conflict detection (multiple automations on same object/event)

**Commit pattern:**
```
feat: /sf-data-model interactive data model designer with limit validation (closes #4)
feat: /sf-automation-map process-to-automation mapping with decision guides (closes #5)

### Agent E: Solution Design & Compliance Skills
**Scope:** `.claude/skills/sf-solution-design/`, `.claude/skills/sf-compliance-check/`

**Issues (in order):**
1. **#6** — /sf-solution-design — pre-build solution design document

### Agent F: Estimation & Story Skills
**Scope:** `.claude/skills/sf-estimate/`, `.claude/skills/sf-user-story/`

**Issues (in order):**
1. **#7** — /sf-estimate — complexity-based scope estimation
   - Create `.claude/skills/sf-estimate/SKILL.md` — Skill that produces complexity-based estimates from requirements. T-shirt sizing (S/M/L/XL) mapped to Salesforce work categories: Configuration, Declarative Automation, Custom Development, Data Migration, Integration, Training & Adoption
   - No time/calendar estimates — complexity and session budgets only
   - Output: complexity matrix, session budget estimate, risk multiplier, minimal/recommended/full scope comparison

2. **#13** — /sf-user-story — conversation notes to formal user stories
   - Create `.claude/skills/sf-user-story/SKILL.md` — Skill that converts unstructured notes/transcripts into formal user stories. Extracts personas, actions, outcomes. Maps to Salesforce capabilities. Standard format: As a [persona] I want [action] So that [outcome] + acceptance criteria + SF implementation notes + complexity + dependencies
   - Batch processes multiple stories from one input
