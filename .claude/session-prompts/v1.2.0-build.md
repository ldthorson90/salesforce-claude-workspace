# v1.2.0 Build Session — Supporting Skills & Operations

## Context
Salesforce Workspace is a multi-project workspace for Salesforce consulting. Read
`CLAUDE.md` for full project context. v1.0.0 shipped the workflow engine and core skills.
v1.1.0 shipped the subagent library and all playbooks. This release adds the remaining
supporting skills, the proposal playbook, and scheduled task automation.

## Release Setup
```
git checkout -b release/v1.2.0 main
```

## Milestone
`v1.2.0` — 6 issues, 4 parallel lanes (A-D). Each lane touches completely different
skill directories. Zero file overlap.

## Agent Team — Launch 4 Agents in Parallel

### Agent A: Client-Facing Skills
**Scope:** `.claude/skills/sf-demo-script/`, `.claude/skills/sf-client-switch/`

**Issues (in order):**
1. **#10** — /sf-demo-script — persona-based demo walkthrough generator
   - Create `.claude/skills/sf-demo-script/SKILL.md` — Generates click-by-click demo scripts for client walkthroughs. Persona-based (role, login context), step-by-step navigation, talking points per step, expected behavior, edge case callouts. Outputs markdown + .docx.

2. **#27** — /sf-client-switch — multi-client context switching
   - Create `.claude/skills/sf-client-switch/SKILL.md` — Loads org alias, project CLAUDE.md, client memories from Obsidian, last git log entries, in-progress workflows. Outputs status summary with current state, open items, active workflows. Must be fast (< 5 seconds).

**Commit pattern:**
```
feat: /sf-demo-script persona-based demo walkthrough generator (closes #10)
feat: /sf-client-switch multi-client context switching (closes #27)
```

### Agent B: Monitoring Skills
**Scope:** `.claude/skills/sf-health-check/`, `.claude/skills/sf-release-notes/`

**Issues (in order):**
1. **#14** — /sf-health-check — proactive org health monitoring
   - Create `.claude/skills/sf-health-check/SKILL.md` — Checks: governor limits (API calls, storage, async queue), metadata hygiene (unused fields, inactive flows, empty permsets), automation health (trigger+flow conflicts, recursion, fault paths), security posture, data quality, technical debt (untested Apex, deprecated APIs, legacy automation). Outputs scorecard with Green/Yellow/Red per category. Supports trend tracking across runs.

2. **#16** — /sf-release-notes — filtered Salesforce release note analysis
   - Create `.claude/skills/sf-release-notes/SKILL.md` — Fetches Salesforce release notes (via web search or Context7), filters by active client clouds/features, identifies breaking changes and deprecations, highlights relevant new capabilities. Uses Ollama (llama3.2) for fast classification. Outputs per-client summary with impact assessment and action items.

**Commit pattern:**
```
feat: /sf-health-check proactive org health monitoring (closes #14)
feat: /sf-release-notes filtered Salesforce release note analysis (closes #16)
```

### Agent C: Cert Prep + Proposal Playbook
**Scope:** `.claude/skills/sf-cert-prep/`, `.claude/playbooks/proposal.yaml`

**Issues (in order):**
1. **#15** — /sf-cert-prep — certification study mode with practice scenarios
   - Create `.claude/skills/sf-cert-prep/SKILL.md` — Supports 9+ certification tracks. Study modes: scenario-based, decision guide walkthrough, feature deep-dive, exam simulation. Pulls Context7 docs for authoritative answers. Uses Ollama (qwen3:8b) for local question generation. Tracks progress in Obsidian.

2. **#24** — /sf-playbook proposal — new business proposal workflow
   - Create `.claude/playbooks/proposal.yaml` — 6-step playbook:
     1. `prospect-intake` → prospect_notes.md
     2. `org-analyzer` (optional) → org_snapshot.md
     3. `capability-assessor` → capability_assessment.md
     4. `scope-estimator` → estimate.md (checkpoint)
     5. `sow-builder` → sow_draft.docx
     6. `proposal-assembler` → proposal.docx (checkpoint before sending)

**Commit pattern:**
```
feat: /sf-cert-prep certification study mode with practice scenarios (closes #15)
feat: /sf-playbook proposal — new business proposal workflow (closes #24)
```
